## E2Eシナリオ追加（4シナリオ）

### E2E-G1: 通知リング連鎖フロー

```
前提: エージェントタスクが実行中
ステップ:
  1. タスクが承認待ちになる
  2. サイドバーのエージェントメニューが黄色パルス
  3. 通知ベルのバッジが+1
  4. 通知ドロップダウンに承認待ちアイテムが追加
  5. macOS通知が表示される（Warning urgency）
  6. エージェント画面に遷移して承認
  7. パルスが停止
  8. タスク完了 → 青パルス（Info）が5秒間表示されて消える

検証: 全4層（サイドバー、タブ、ベル、macOS）が連動
```

### E2E-G2: ブラウザ + コンテキスト連携フロー

```
前提: TestRepo に設計書 + PR（MockGitHubGateway）
ステップ:
  1. PR一覧画面でPRをクリック
  2. ブラウザパネルでPRページが開く
  3. BrowserContextBar に影響設計書2件が表示
  4. 設計書リンクをクリック → 設計書ビューワーが分割ペインで開く
  5. 「AIレビュー実行」ボタン → レビュー結果が下部ペインに表示

検証: ブラウザ → doc-mapping → レビューが自然に連鎖
```

### E2E-G3: Socket API外部連携フロー

```
前提: DevNest起動中、Socket APIサーバー稼働
ステップ:
  1. 外部プロセスからSocket APIに接続
  2. "notify" メソッドで通知送信 → 通知ベルに反映
  3. "task.submit" でタスク登録 → エージェントキューに追加
  4. "docs.staleness" で鮮度取得 → JSONレスポンス確認
  5. "browser.open" でPRを開く → ブラウザパネルが表示

検証: 外部プロセスからDevNestの全機能が操作可能
```

### E2E-G4: 分割ペインでのマルチタスク作業

```
前提: プリセット "code-review" を選択
ステップ:
  1. 左上ペイン: PRブラウザが開く
  2. 右上ペイン: 影響設計書が自動で開く
  3. 下部ペイン: AIレビュー結果が表示
  4. 分割バーをドラッグしてサイズ変更
  5. アプリを再起動
  6. 同じレイアウトが復元される

検証: プリセット、リサイズ、永続化が動作
```
